Git Basics

Install Git

Git comes standard on all Macs. If you're running Windows, you'll have to intall git yourself first. To check what version of git you have installed, run this helpful command:

git --version

Create a Repository

A repository is simply a location where all your project files are stored. Git allows us to version control these files, to allow for a cleaner workflow. There are two ways you can create a repository with git.

Create a repository from scratch

Let's say you want to create a new project called "timeless_nightfall". To get started, all you need to do is type this command:

git init timeless_nightfall

A new folder will be created in your chosen directory with the name "timeless_nightfall".

Create a repository from an existing folder

What if you had already started the project? No worries. You can easily convert any exisiting project folder to a git repository by running this command. (Just be sure you are in that project's home directory).

git init

Note:

Using the git init command will create a hidden .git file in your folder. To see this and other hidden files, run ls -a.

Create a File

Let's make a quick addition to our project. First, let's make sure we're in our project file's home directory. Let's use the project we just created.

cd timeless_nightfall

cd stands for "Change Directory". We're telling our terminal to take us into the project folder "timeless_nightfall".

Now that we're in our project's home directory, let's create a readme file.

touch README.md

Pushing up changes to the repository

Now that we've created a new file, we're ready to push that up and make it available to everyone on the team.

Check the status

It's always good practice to check the status of your changes. Running this show us what files we've altered.

git status

Remove a File

Let's say we no longer wanted that README.md file for some reason. Using this command, you can easily remove any file right from your terminal.

rm -r README.md

Stage files

Let's say we didn't remove that file. We put a lot of work into this, and actually want to get this pushed up to our team. There are a few ways you can add, or stage, files to be comitted.

Add a single file

git add README.md

If you had multiple changes to your project, you can just add them all at once.

Add all files

git add .

Commit

Now that you have your files staged, you can commit them to be pushed. You'll need to make sure to add a message so others know what changes you made.

git commit -m "Added README file"

Push

You're ready to push! Type this command to send it up.

git push

Pull

Because other teammates may be working on the same project, you'll want to make sure to pull in their changes. This will make sure you're always up to date.

git pull
