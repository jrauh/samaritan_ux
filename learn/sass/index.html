Sass Setup

Install Sass

If you're on a Mac, installing sass is extremely easy and can be done in just a few steps. Open your terminal and type in the following commands to install the Ruby Gem:

gem install sass
sudo gem install sass
sass -v

Set up your project

Before you start using sass, it may be helpful to install git, which will allow you to quickly and efficiently manage your project files. Here are a few quick commands to get you going:

git init new-project
cd new-project
touch index.html styles.css styles.scss

Compile sass to css

Now that you have your files ready to go, you can start editing your stylesheet. Keep in mind, all of your edits will be done through your .scss file, never in your .css file. Run this command to compile your sass into css, then open your index file to view your changes.

sass --watch input.scss:output.css
open index.html

The Basics

Nesting & Parent Selectors

Nesting within Sass allows for a cleaner interface, as it helps avoid having to consistently type out the name of your parent selector. If you ever need to specifically target where the parent selector should be placed, the & symbol will allow you to determine that placement.

  <div class="card small">
      <h3 class="title">
        Title
      </h3>
    </div>

  .card {
      background-color: #FFF;
      width: 15rem;

      &:hover {
        opacity: .8;
      }

      &.small {
        width: 10rem;
      }

      .title {
        font-size: 1.2rem;
      }
    }

  .card {
      background-color: #FFF;
      width: 15rem;
    }

    .card:hover {
      opacity: .8;
    }

    .card.small {
      width: 10rem;
    }

    .card .title {
      font-size: 1.2rem;
    }

Variables

Variables offer a quick shortcut that can be reused throughout your stylesheet. By referencing the $ symbol, you are able to create a substitute that can be utilized anywhere within your project.

  $primary-color: #6A8DD3;

    .card {
      border-color: $primary-color;
    }

  .card {
      border-color: #6A8DD3;
    }

Mixins

One easy way to use the @mixin directive is to apply it for vendor prefixed elements. To create a mixin, simply define a mixin name and a variable, which will be passed to each vendor prefix. This can now be referenced anywhere within your project by using an @include, followed by the mixin name and a desired value.

  @mixin user-select($value) {
      -webkit-user-select: $value;
      -moz-user-select: $value;
      -ms-user-select: $value;
      -o-user-select: $value;
      user-select: $value;
    }

    .card {
      @include user-select(none);
    }

  .card {
      -webkit-user-select: none;
      -moz-user-select: none;
      -ms-user-select: none;
      -o-user-select: none;
      user-select: none;
    }

Extend

The @extend function allows one selector to inherit the styles of another selector. This can be especially useful when one class utilizes all of another class's styles, along with some of its own specific styles.

  /* Sass Input */
    .column {
      padding: 1rem;
      float: left;
      margin: 0;
      width: 100%;
    }

    .column-half {
      @extend .column;
      width: 50%;
    }

  /* CSS Output */
    .column {
      padding: 1rem;
      float: left;
      margin: 0;
      width: 100%;
    }

    .column-half {
      padding: 1rem;
      float: left;
      margin: 0;
      width: 100%;
      width: 50%;
    }
